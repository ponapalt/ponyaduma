ぽにゃづま 新機能プラン
========================

■ 実装済み
----------

1. ユーザー名変更機能
   - メインメニューに「名前を教えて」追加 (menu.txt)
   - あやりりすの名前入力起動('メニュー')を利用
   - 空欄/空欄繰り返し/敬称確認/完了トークを実装
   - 敬称定義・初期名はaya_lilith_config.dicの既存設定を流用

2. 曜日トーク・季節トーク
   - 曜日: 月～日の7種を ランダムトーク○曜 : array で追加 (aitalk.txt)
   - 季節: 春夏秋冬の4種を ランダムトーク春 : array 等で追加 (aitalk.txt)
   - あやりりすのGetEvalTalkArrayが自動でスロットに乗せてくれる


■ 未実装（優先度高）
--------------------

5. 起動回数の節目トーク
   - ghostuptimes（累積起動回数）を利用
   - 10回/50回/100回/500回/1000回 等の節目で特別トーク
   - 課題: あやりりす側の起動トーク選択(BootCloseEx)は配列からランダム選択の仕組み。
     節目トークを「必ず出す」にはBootCloseExの前段で割り込む仕組みが必要。
     候補:
     (a) AYL_OnBootを辞書側でオーバーライド → あやりりすの更新で壊れるリスク
     (b) あやりりすに「起動時優先トーク」的なフックを新設 → あやりりす改修が必要
     (c) 起動カスタムに入れて混ざる形で妥協 → 節目なのに出ない可能性がある
   → ぽなさん側であやりりす改修策を検討中


■ 未実装（優先度低）
--------------------

3. 特定ゴーストとのコミュニケーション（OnCommunicate）
   - 他のぽなパロディゴーストとの掛け合い
   - change.txtに個別ゴースト反応は既にあるので、その延長で

4. 選択肢ベースの会話機能
   - 自由入力キーワードマッチはNG（LLM時代に嘘くさい）
   - メニューから選ぶ形式で実装する
   - 例: 「ぽにゃに聞きたいこと」→「好きなもの」「苦手なもの」「最近どう？」等
   - ユーザーの前だけ見せる「素のぽにゃ」を活かす設計

6. 天気/カレンダー連動
   - SSP側の実装待ち


■ メモ
------
- 文法チェック: ./tamac.exe "ghost/master/yaya.dll"
- 名前入力敬称はaya_lilith_config.dicで定義済み。辞書側で重複定義するとエラーになる
- あやりりすの時間スロット命名規則:
  ランダムトーク○曜 / ランダムトーク春 / ランダムトーク3月 / ランダムトーク朝 等
  起動・終了も同様: 朝に起動 / 起動月曜 / 起動2月25日 等
