## トーク実装の指針

### ネタ選び

- **笑いのツボはぽにゃのキャラ**（板挟み・目が死ぬ）であって技術知識ではない。専門的な状況は日常あるあるに翻訳する（例：「犯人は過去の自分」「自動化に3日かけて手作業より遅い」「最終版_v2_修正_本当の最終版」）
- **伺か・SSP固有ネタは残す**が、汎用開発ネタは日常あるあるに置換済みのもののみ
- **スラップスティックは視覚的イメージで強化**。「ブーメラン顔面直撃」「もぐらたたき」等、読んで絵が浮かぶ表現
- **ぽにゃの空回りは「考えすぎ」が根っこ**。板挟みだけでなく、入力待ちフリーズ（雑談が無理）、処理遅延（完璧な返しが翌日届く）、入力不足のまま無理に出力して盛大にズレる、完璧な返しを作ったのに飲み込んで終わる——等、「考えすぎて壊れる」バリエーションで展開できる
- **挙動がLLM的になるのは結果であってコンセプトではない**。あくまで「考えすぎる子」として描く

### 執筆ルール

- **迷ったらuser(ぽな本人)に質問責め**。セルフパロディなので推測より本人に聞く方が速い。遠慮不要
- **見せ場は掛け合い、機能通知はぽにゃ単発**。ランダムトーク・起動終了・切替・消滅系は掛け合い。ネットワーク更新等はぽにゃだけで十分
- **消滅系は例外的にシリアス可**。切なさを積んでオチで着地。蟲ゅうが素直になれる数少ない場面
- **蟲ゅうの素直な肯定は切り札**。「ぽにゃはぽにゃのままでええ」はここぞの場面だけ
- **決め台詞**：「たぶん」「最初からわかってたのです」「バランス」は繰り返し可だが同一トーク内で重複しない
- **オチの型を散らす**。「最初からわかってたのです」「たぶん」「聞こえないのです」等が連続しないよう配列全体で意識
- **時報は生活リズムの流れ**。深夜→徹夜、朝→寝不足、午後→眠気、夕方→やり残し
- **（ユーザー）への語りかけ**：ぽにゃが照れ・不安 → 蟲ゅうが軽く肯定して着地
- **猫耳・しっぽ**は椅子に挟む等「持て余す日常の一部」として。萌え記号にしない
- **艦これ由来の表現はほどほどに**。ぽにゃ自身の言葉に置き換え、匂わせで残す程度

### 笑いの技法（上方漫才・落語ベース）

- **天丼**：同じ構造のボケを2～3回エスカレーション
- **段取り→崩壊**：丁寧に積み上げてから一気に崩す
- **間**：`ぽ０６：…` + `改行無し` で沈黙→爆発。蟲ゅうの沈黙も対比として有効
- **ずらし**：蟲ゅうが予想外の角度から返す（褒め＋ズレの二段構え）

### 禁則・校正の原則

- **蟲ゅうが答え・正解・助言を出してはいけない**。「堅実って大事やろ」等はNG。笑うか聞くかまで
- **ぽにゃが自分で解決策を出して着地するパターンも減らす**。「ドキュメントを充実させるのです」→予定調和。詰んだまま・笑いで流す方がこの二人らしい
- **オチがついたら即終わる**。蛇足の掛け合いを切るだけでテンポが劇的に改善する。蟲ゅうの短い一言で締めるのが最も切れ味がよい
- **類似パターンの重複を間引く**。同じ構造のオチが複数ある場合、最も切れ味のよい1本だけ残す
- **三点リーダーは「…」（1個）に統一**
- ぽにゃを「無自覚な毒舌キャラ」にしない（自覚して飲み込んでる）
- ぽにゃを「上から目線」にしない（聞く側のキャラ）
- 板挟みの重さ・深刻さを前面に出さない（楽しさ優先。消滅系のみ例外）

## あやりりすEX記法

SakuraScriptの簡易記法変換エンジン。トークをべた書きで記述でき、スコープ指定・サーフィス変更・\eなどを自動生成する。

### このゴーストのEXヘッダ設定（aya_lilith_ex_config.dic）

- `ぽ`/`ポ` = \0側（本体）
- `れ`/`レ` = \1側
- 標準表情: \0側=surface0, \1側=surface10

### 基本構文

```
ヘッダ＋サーフィスID：テキスト
```

例：
- `ぽ００：なのです。` → `\0\s[0]なのです。\e`
- `ぽ０５：バグ大好き♪` → `\0\s[5]バグ大好き♪\e`
- `れ１０：やれやれ。` → `\1\s[10]やれやれ。\e`

### ヒアドキュメント

- `<<" ">>` : 通常のトーク用。YAYA変数 `%(...)` が展開される。
- `<<' '>>` : ランダムトーク配列用（性能最適化）。YAYA変数は展開されない。

```yaya
// 単発トーク（<<" ">>を使用）
シェル変更前
{
	<<"
	ぽ００：改装するのです。
	">>
}

// ランダムトーク配列（<<' '>>を使用、空行で区切り）
ランダムトーク : array
{
	<<'
	ぽ０５：バグ大好き♪
	'>>
}
```

### 複数行の会話

サーフェスを変えずにトークを続ける場合はサーフェス番号を省略できる
ヘッダなしの行は前の発言者の続き

```yaya
<<'
ぽ０５：ここにおもしろいボケが入ります。
：サーフェスを変えずに改行して追加で話します。
れ１０：ここに素晴らしいツッコミが入ります。
ぽ０１：ここにオチをつけます。
'>>
```

### ウエイト自動挿入

句読点・記号に自動でウエイトが入る（「、」「…」→\w5、「。」「！」「？」「」」→\w9）。手動で`\w`タグを書く必要はない。

### 接頭辞

同一行内でサーフィスだけ変えたい場合は`改行無し`接頭辞を使う。`\s[N]`タグの直書きは不要。

```yaya
ぽ０５：やるのです！……
改行無しぽ０４：いつか。
```

他の接頭辞サンプル：

待ち無しぽ０５： //\w3　「待無」でも可
待ちゼロぽ０６： //まったくなし　「待０」でも可
待ち多めぽ０８： //\w9 x 4　「待多」でも可
改行前ぽ０３： //改行・ウエイト前に表情を変える

`待ち多め`は「衝撃の事実を明かす前のタメ」に効果的。例：
```
ぽ０６．２０２５：机の上を片付けてたら、3ヶ月前のメモが出てきたのです。
れ１０：何て書いてあったん。
ぽ０４：「重要！あとで絶対やること！」
れ１０：で、何をやるん。
ぽ０６：…
改行無しぽ０４．２０３５：何をやるのか書いてなかったのです。
```

### ユーザー名

- あやりりすEX記法内: `（ユーザー）`（全角括弧）を使う。<<' '>>内でも動作する。
- SakuraScript記法内: `%(username)` を使う。

### フラグ操作との併用

変数代入の後にヒアドキュメントを置くと、代入が実行されてからトークが返される。

```yaya
Menu_TALKMAX
{
	aitalkinterval = 15
	<<"
	ぽ００：たくさん話すのです。
	">>
}
```

### 変換対象の判断基準

- アルゴリズムなし、または単純なフラグ操作+トークのみの関数 → EX記法に変換
- if/elseif分岐、メニュー構築（メニュー()関数）、文字列連結（--）、特殊タグ（\_q, \b, \6等）を含む関数 → SakuraScript記法のまま維持
