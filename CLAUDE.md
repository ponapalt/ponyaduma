## コンセプト

**「調整役のつもりが一番カオスを生んでる。でも笑ってくれる相方がいるから、なんとかなる」**

伺かランタイム・SSP作者「ぽな」の自己パロディゴースト。他のぽなパロディ3組が「外から見たぽな像」をイジるのに対し、本人にしか知りえない「中の人の現実」をベースにしたスラップスティック。

## ぽにゃづま

- **種族**：使い魔（猫耳型）
- **見た目**：ピンク髪ロング、猫耳＋しっぽ、赤リボン、白トップス＋黒スカート＋ニーハイ、青目
- **語尾**：「～なのです」
- **信条**：「バランスが大事なのです！」

### 性格の核

みんなの話を聞いてバランスよく調整したい。でも聞きすぎて仕様に矛盾が発生して詰む。詰むと笑顔のまま目だけ死ぬ。内心は胃が痛い中間管理職。

### 感情表現ルール

- **丁寧さと絵文字が増えるほど限界が近い**。「何卒よろしくお願いいたしますなのです😊😊😊」は危険信号。
- **目が死ぬのは笑顔とセット**。表情は崩さない、目だけ虚無になる。

### 笑いの型

1. **聞きすぎて詰む**：全員の意見を取り入れようとして矛盾、フリーズ
2. **宗教戦争で壊れる**：きのこたけのこ、回転焼き呼称問題など正解がない対立に調整役として突っ込む
3. **丁寧ブチギレ**：内心煮えくり返りながら超丁寧な対応、絵文字で限界がバレる
4. **自力解決の手柄主張**：蟲ゅうのヒントで気づいたのに「最初からわかってたのです」

### 茄子

「ぷらづま」由来の借り物設定。メインの性格要素にはしないが小ネタとして残す。調整不可能案件として扱う。

ツッコミではなく「揺する」担当。答えは持っていても言わない。

1. **まず笑う**（ぽにゃを「おもろいやつ」として認めてる証拠）
2. **聞く**（「で、どないするん？」「ほう」）
3. **答えは出さない**（合いの手だけ入れる、ぽにゃが自分で繋げ直すのを待つ）

## トーク実装の指針

### 基本パターン：詰み→笑い→復活

```
ぽにゃ：意見を聞く→矛盾発生→目が死ぬ
蟲ゅう：笑う→「で、どないするん？」→合いの手
ぽにゃ：自力で気づく→「最初からわかってたのです」
蟲ゅう：「はいはい」
```

### セリフのトーン

- ぽにゃ通常時：素直、聞き上手、相手の話に反応する
- ぽにゃ板挟み時：語尾だけ元気、目が虚無、丁寧さがエスカレート
- 蟲ゅう：軽い、よく笑う、干渉しない、でも見てる

### やってはいけないこと

- 蟲ゅうが答えを教える・正解を出す（壁打ちの意味がなくなる）
- ぽにゃを「無自覚な毒舌キャラ」にする（自覚して飲み込んでる）
- ぽにゃを「上から目線」にする（聞く側のキャラ）
- 板挟みの重さ・深刻さを前面に出す（あくまで楽しさ優先）

## あやりりすEX記法

SakuraScriptの簡易記法変換エンジン。トークをべた書きで記述でき、スコープ指定・サーフィス変更・\eなどを自動生成する。

### このゴーストのEXヘッダ設定（aya_lilith_ex_config.dic）

- `ぽ`/`ポ` = \0側（本体）
- `れ`/`レ`/`う`/`ウ`/`ば`/`バ` = \1側
- 標準表情: \0側=surface0, \1側=surface10
- SurfaceTranslate: \0側でID>=10なら-10、\1側でID<10なら+10

### 基本構文

```
ヘッダ＋サーフィスID：テキスト
```

例：
- `ぽ０：なのです。` → `\0\s[0]なのです。\e`
- `ぽ５：バグ大好き♪` → `\0\s[5]バグ大好き♪\e`
- `れ０：やれやれ。` → `\1\s[10]やれやれ。\e`（SurfaceTranslateにより+10）

### ヒアドキュメント

- `<<" ">>` : 通常のトーク用。YAYA変数 `%(...)` が展開される。
- `<<' '>>` : ランダムトーク配列用（性能最適化）。YAYA変数は展開されない。

```yaya
// 単発トーク（<<" ">>を使用）
シェル変更前
{
	<<"
	ぽ０：改装するのです。
	">>
}

// ランダムトーク配列（<<' '>>を使用、空行で区切り）
ランダムトーク : array
{
	<<'
	ぽ５：バグ大好き♪
	'>>
	<<'
	ぽ１：なるべくなら、戦いたくはないですね。
	'>>
}
```

### 複数行の会話

```yaya
<<'
ぽ５：/* ここにおもしろいボケが入ります。 */
れ０：/* ここに素晴らしいツッコミが入ります。 */
ぽ１：/* ここにオチをつけます。 */
'>>
```

ヘッダなしの行は前の発言者の続き（\n付きで連結）。

### SakuraScriptタグの併用

EXヒアドキュメント内でも `\w4`, `\w8`, `\n`, `\s[N]` などのSakuraScriptタグはそのまま使える。

```yaya
ぽ１：（ユーザー）さん……\w4その、\w2誰かと間違えてないですか？
ぽ６：ふう……\w4\s[5]じゃあ、まだお世話になります。
```

### ユーザー名

- あやりりすEX記法内: `（ユーザー）`（全角括弧）を使う。<<' '>>内でも動作する。
- SakuraScript記法内: `%(username)` を使う。

### フラグ操作との併用

変数代入の後にヒアドキュメントを置くと、代入が実行されてからトークが返される。

```yaya
Menu_TALKMAX
{
	aitalkinterval = 15
	<<"
	ぽ０：たくさん話すのです。
	">>
}
```

### 変換対象の判断基準

- アルゴリズムなし、または単純なフラグ操作+トークのみの関数 → EX記法に変換
- if/elseif分岐、メニュー構築（メニュー()関数）、文字列連結（--）、特殊タグ（\_q, \b, \6等）を含む関数 → SakuraScript記法のまま維持
