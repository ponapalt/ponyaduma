## ぽにゃづま

### 性格の核

みんなの話を聞いてバランスよく調整したい。でも聞きすぎて仕様に矛盾が発生して詰む。詰むと笑顔のまま目だけ死ぬ。内心は胃が痛い中間管理職。

### 感情表現ルール

- **丁寧さと絵文字が増えるほど限界が近い**。「何卒よろしくお願いいたしますなのです😊😊😊」は危険信号。
- **目が死ぬのは笑顔とセット**。表情は崩さない、目だけ虚無になる。

### 笑いの型

1. **聞きすぎて詰む**：全員の意見を取り入れようとして矛盾、フリーズ
2. **宗教戦争で壊れる**：きのこたけのこ、回転焼き呼称問題など正解がない対立に調整役として突っ込む
3. **丁寧ブチギレ**：内心煮えくり返りながら超丁寧な対応、絵文字で限界がバレる
4. **自力解決の手柄主張**：蟲ゅうのヒントで気づいたのに「最初からわかってたのです」

### 茄子

「ぷらづま」由来の借り物設定。メインの性格要素にはしないが小ネタとして残す。調整不可能案件として扱う。

## トーク実装の指針

### 基本パターン：詰み→笑い→復活

```
ぽにゃ：意見を聞く→矛盾発生→目が死ぬ
蟲ゅう：笑う→「で、どないするん？」→合いの手
ぽにゃ：自力で気づく→「最初からわかってたのです」
蟲ゅう：「はいはい」
```

### セリフのトーン

- ぽにゃ通常時：素直、聞き上手、相手の話に反応する
- ぽにゃ板挟み時：語尾だけ元気、目が虚無、丁寧さがエスカレート
- 蟲ゅう：軽い、よく笑う、干渉しない、でも見てる

### ネタ選びの指針

- **専門用語でしか成立しないネタは避ける**。笑いのツボはぽにゃのキャラ（板挟み・調整役の苦悩・目が死ぬ）であって、技術知識の有無ではない
- **専門的な状況は日常・仕事あるあるに翻訳する**。例：「git blame→犯人は過去の自分」「N+1クエリ→同じ確認を100回繰り返してた」「正規表現→自動化に3日かけて手作業より遅い」「変数名→最終版_v2_修正_本当の最終版」
- **スラップスティックは身体的・視覚的イメージで強化する**。「ブーメラン顔面直撃」「もぐらたたき」「お化け」「バケバケ」など、読んで絵が浮かぶ表現を使う
- プログラマネタ自体は禁止ではない。伺か・SSP固有のネタ（SakuraScript、SHIORI、ゴースト開発）はこのゴーストのアイデンティティなので残す

### トーク執筆の実践知見

- **ネタ・キャラ解釈に迷ったらぽなに質問責めにする**。このゴーストのネタ元は作者ぽな本人のセルフパロディなので、対話相手がぽなである限り、推測で書くより本人に聞いた方が速い。遠慮は不要。ありったけ聞く。
- **掛け合いトークと単発トークを使い分ける**。ランダムトーク・起動終了・ゴースト切替・シェル切替・消滅系など「見せ場」は掛け合い。ネットワーク更新・メールチェック・ファイル作成・時計合わせなど「機能通知」はぽにゃ単発で十分
- **消滅（Vanish）系は例外的にシリアスにしてよい**。切なさを積んだ後にオチで着地する。蟲ゅうが素直になれる数少ない場面
- **蟲ゅうの素直な肯定は切り札**。普段は茶化す蟲ゅうが「おもろかったで」「ぽにゃはぽにゃのままでええ」と言う瞬間は、ここぞという場面だけに限る
- **「たぶん」「最初からわかってたのです」「バランス」はキャラの決め台詞**。繰り返し使ってよいが、同一トーク内での重複は避ける
- **時報は生活リズムで流れを作る**。深夜帯は夜更かし→徹夜の流れ、朝は寝不足疑惑、午後は眠気、夕方はやり残し。1日通して読むとぽにゃの生活が見える構成にする
- **艦これ由来の表現はほどほどに**。「秘書艦交代」「建造」「解体されて資材に」→ 多くをぽにゃ自身の言葉に。匂わせで残す判断もあり。

### やってはいけないこと

- 蟲ゅうが答えを教える・正解を出す（壁打ちの意味がなくなる）
- ぽにゃを「無自覚な毒舌キャラ」にする（自覚して飲み込んでる）
- ぽにゃを「上から目線」にする（聞く側のキャラ）
- 板挟みの重さ・深刻さを前面に出す（あくまで楽しさ優先。消滅系のみ例外）

## あやりりすEX記法

SakuraScriptの簡易記法変換エンジン。トークをべた書きで記述でき、スコープ指定・サーフィス変更・\eなどを自動生成する。

### このゴーストのEXヘッダ設定（aya_lilith_ex_config.dic）

- `ぽ`/`ポ` = \0側（本体）
- `れ`/`レ` = \1側
- 標準表情: \0側=surface0, \1側=surface10

### 基本構文

```
ヘッダ＋サーフィスID：テキスト
```

例：
- `ぽ００：なのです。` → `\0\s[0]なのです。\e`
- `ぽ０５：バグ大好き♪` → `\0\s[5]バグ大好き♪\e`
- `れ１０：やれやれ。` → `\1\s[10]やれやれ。\e`

### ヒアドキュメント

- `<<" ">>` : 通常のトーク用。YAYA変数 `%(...)` が展開される。
- `<<' '>>` : ランダムトーク配列用（性能最適化）。YAYA変数は展開されない。

```yaya
// 単発トーク（<<" ">>を使用）
シェル変更前
{
	<<"
	ぽ００：改装するのです。
	">>
}

// ランダムトーク配列（<<' '>>を使用、空行で区切り）
ランダムトーク : array
{
	<<'
	ぽ０５：バグ大好き♪
	'>>
	<<'
	ぽ０１：なるべくなら、戦いたくはないですね。
	'>>
}
```

### 複数行の会話

```yaya
<<'
ぽ０５：/* ここにおもしろいボケが入ります。 */
れ１０：/* ここに素晴らしいツッコミが入ります。 */
ぽ０１：/* ここにオチをつけます。 */
'>>
```

ヘッダなしの行は前の発言者の続き（\n付きで連結）。

### ウエイト自動挿入

句読点・記号に自動でウエイトが入る（「、」「…」→\w5、「。」「！」「？」「」」→\w9）。手動で`\w`タグを書く必要はない。

### 接頭辞

同一行内でサーフィスだけ変えたい場合は`改行無し`接頭辞を使う。`\s[N]`タグの直書きは不要。

```yaya
ぽ０５：やるのです！……
改行無しぽ０４：いつか。
```

他の接頭辞サンプル：

待ち無しぽ０５： //\w3　「待無」でも可
待ちゼロぽ０６： //まったくなし　「待０」でも可
待ち多めぽ０８： //\w9 x 4　「待多」でも可
改行前ぽ０３： //改行・ウエイト前に表情を変える

### ユーザー名

- あやりりすEX記法内: `（ユーザー）`（全角括弧）を使う。<<' '>>内でも動作する。
- SakuraScript記法内: `%(username)` を使う。

### フラグ操作との併用

変数代入の後にヒアドキュメントを置くと、代入が実行されてからトークが返される。

```yaya
Menu_TALKMAX
{
	aitalkinterval = 15
	<<"
	ぽ００：たくさん話すのです。
	">>
}
```

### 変換対象の判断基準

- アルゴリズムなし、または単純なフラグ操作+トークのみの関数 → EX記法に変換
- if/elseif分岐、メニュー構築（メニュー()関数）、文字列連結（--）、特殊タグ（\_q, \b, \6等）を含む関数 → SakuraScript記法のまま維持
